# 16. Search
## SEARCH

* 문서 버전: v1.0
* 작성일: 2026-04-16
* 참조: `docs/01-product/functional-spec.md § SEARCH`

---

### 1. 기능 목적

* 맵 내 노드를 **텍스트 또는 태그로 빠르게 탐색**하는 기능
* 검색 결과 노드로 캔버스 즉시 이동 (Center Node)
* 대형 맵에서 원하는 노드를 신속하게 찾을 수 있는 UX 제공

---

### 2. 기능 범위

* 포함:
  * 텍스트 검색 — 노드 content 키워드 매칭
  * 태그 검색 — 특정 태그가 붙은 노드 목록
  * 검색 결과 하이라이트
  * 검색 결과 노드로 캔버스 이동 (Center Node)
  * 실시간 검색 (입력 즉시 결과 업데이트)

* 제외:
  * 전체 맵 대상 글로벌 검색 (맵 내 검색만)
  * 정규식 검색
  * note 필드 검색 (후순위)

---

### 3. 세부 기능 목록

| 기능ID      | 기능명      | 설명                       | 주요 동작             |
| --------- | -------- | ------------------------ | ----------------- |

| SEARCH-02 | 태그 검색    | 특정 태그 노드 목록              | 태그 선택             |

### 4. 기능 정의 (What)

#### 4.1 검색 UI 구조

```text
┌──────────────────────────────────────┐
│ 🔍 검색어 입력...             [✕]    │
├──────────────────────────────────────┤
│ 결과 3개                              │
│  📄 Apache 설치                       │
│  📄 MySQL 설정                        │
│  📄 서버 보안 설정                     │
└──────────────────────────────────────┘
```

#### 4.2 텍스트 검색 동작

* 클라이언트 사이드 필터링 (맵 로딩 후 메모리 내 검색)
* 대소문자 구분 없음, 부분 일치 지원
* 입력 변경 시 즉시 결과 갱신 (debounce 300ms — 대형 맵 한정)

---

### 5. 동작 방식 (How)

#### 5.1 사용자 동작

* `Ctrl+F` 또는 툴바 검색 아이콘 → 검색 패널 열기
* 키워드 입력 → 실시간 결과 표시
* 결과 클릭 → 해당 노드 Center Node 이동
* `↑↓` 방향키 → 결과 목록 순회
* `Enter` → 선택 결과로 이동
* `ESC` → 검색 닫기, 전체 노드 복원

---

#### 5.2 시스템 처리

* 검색 입력 → Document Store 전체 노드 content 클라이언트 필터링
* 결과 노드 ID → 하이라이트 (나머지 dim)
* 결과 클릭 → `viewport.centerNode(nodeId)` 호출

---

#### 5.3 표시 방식

* 검색 결과 노드: 강조 테두리, 정상 불투명도
* 비검색 노드: 30% 투명도
* 결과 0개: "검색 결과가 없습니다" 표시

---

### 6. 규칙 (Rule)

* 최소 검색어 1자 이상
* 최대 결과 표시 100개
* collapsed 노드의 숨겨진 자식도 검색 대상
* 검색 중 캔버스 편집 가능 (상태 유지)

#### 6.1 MVP 검색 범위

- 대상:
  - `nodes.text`
  - `tags.name` / `node_tags`
- 제외:
  - `node_notes.content` (후순위)
  - 첨부파일 본문 OCR
  - 오디오/비디오 transcript

#### 6.2 구현 단계

1. MVP: 클라이언트 필터링 + 서버 기본 LIKE/ILIKE
2. V2: PostgreSQL Full Text Search + GIN index
3. V3: note / attachment metadata / AI semantic search 확장

---

### 7. 예외 / 경계 (Edge Case)

* **빈 검색어**: 전체 노드 표시
* **결과 0개**: 안내 메시지
* **1000+ 노드**: debounce 300ms 적용

## MVS 구현 — 검색 결과 강조 + 뷰어 검색 (2026-07)

- **에디터**: 검색 패널에서 결과를 클릭하면 해당 노드를 **노란
  채움(#FFE066) + 붉은 테두리(#DC2626, 2.6px)** 로 어떤 스타일보다
  우선해 또렷하게 표시하고(interactionStore.searchHitId), **접힌
  조상을 펼친 뒤 해당 노드를 화면 중앙 + 100% 보기**로 이동한다
  (2026-07: documentStore.expandAncestors +
  viewportStore.requestCenterNode — 이전에는 강조만 하고 맵이
  움직이지 않았다). 캔버스에서 다른 노드를 클릭/조작하면 해제.
- **강조 노드 글자색 = 진한 고정색(#1F1B16)** (2026-07): 다크
  모드에서 밝은 글자가 노란 배경에 묻혀 내용을 읽을 수 없던 문제 —
  에디터(NodeRenderer)·뷰어(drawNode) 동일.
- **HTML 내보내기 뷰어에도 검색 추가 — 에디터 검색 패널과 동일한
  인터페이스** (2026-07 갱신): 헤더 검색창(placeholder "노드 · 태그 ·
  노트 검색", 🔍 아이콘)에 입력하면 **결과 목록 드롭다운**이 뜬다 —
  "결과 N건" + 항목(제목의 일치 구간 mark 강조 · 상위 경로 › 표기 ·
  일치 위치 노드/태그/노트/링크). 결과 클릭 = 노란+붉은 강조 + 접힌
  조상 펼침 + **화면 중앙 + 100% 보기** 이동(에디터와 동일), 클릭
  후에는 **결과 목록이 닫혀** 맵(강조 노드)을 가리지 않는다 — 검색창을
  다시 클릭하면 목록이 다시 열린다(강조·선택 표시 유지).
  Enter = 다음 결과 순회, Esc/비우면 해제, 바깥 클릭 = 목록 닫기
  (강조 유지). 검색 대상도 에디터와 동일(노드·태그·노트·링크).
  Focus 모드 중 검색하면 전체 맵으로 복귀 후 탐색.
- **뷰어 강조 해제 경로** (2026-07 — 에디터와 동일): ① 맵의 다른
  노드 클릭 ② 빈 캔버스 클릭(선택도 함께 해제, Pan 드래그 직후의
  click은 제외) ③ 결과 목록에서 같은 항목 재클릭(토글) ④ Esc/검색어
  비우기.
