### 1. 기능 목적

* 맵 편집 중 **AI를 활용하여 노드 생성·확장·요약·태스크 추출**을 자동화하는 기능
* Claude / GPT 등 LLM 기반으로 마인드맵 구조 자동 생성 및 확장
* 협업 스레드 요약과 할일 노드 생성으로 회의·논의 결과를 즉시 구조화

### 2. 기능 범위

* 포함:
  * AI Mindmap 자동 생성 (AI-01)
  * 선택 노드 기준 AI 확장 (AI-02)
  * Node Thread 핵심 논점 요약 (AI-03)
  * 댓글에서 Action Item 추출 (AI-04)
  * 승인된 태스크 후보를 자식 TODO 노드로 생성 (AI-05)

* 제외:
  * AI Workflow (→ `19-ai-workflow.md`)

### 4. 기능 정의 (What)

#### 4.1 AI-01 Mindmap 자동 생성

```text
┌──────────────────────────────────────────────────────┐
│  AI Mindmap 생성                                      │
│  주제 또는 설명을 입력하세요:                              │
│  ┌────────────────────────────────────┐               │
│  │ Linux 서버 구축 절차                │               │
│  └────────────────────────────────────┘               │
│  [ 생성하기 ]                                          │
└──────────────────────────────────────────────────────┘
```

* 프롬프트 → `POST /ai/generate` → `ai_jobs` row INSERT (status: pending)
* BullMQ Worker가 LLM 호출 → Markdown outline 반환
* Markdown outline → Node Tree 파싱 → Document Store 반영

#### 4.2 AI-02 노드 확장

* 선택 노드 우클릭 > `AI 확장` 또는 툴바 AI 버튼
* 선택 노드 text + 상위 2레벨 컨텍스트 → LLM 전송
* 반환된 자식 아이디어 → 선택 노드 하위에 신규 자식 노드 생성

#### 4.3 AI-03 Thread 요약

* Node Thread 패널 상단 `[AI 요약]` 버튼
* Thread 메시지 전체 → LLM 요약 프롬프트
* 결과: 핵심 논점 / 결정 사항 / 미결 사항 3구분 출력
* 요약 결과는 노드 Note에 저장하거나 별도 팝업으로 표시

#### 4.4 AI-04 태스크 추출

* Thread 요약과 연계 또는 독립 실행
* 댓글 텍스트 분석 → Action Item · 담당자 후보 · 기한 후보 목록 반환
* 사용자 검토 후 개별 승인/거부 선택

#### 4.5 AI-05 태스크 노드 생성

* 승인된 태스크 후보 → 선택 노드의 자식 TODO 노드 일괄 생성
* 노드 text = 태스크명, note = 기한·담당자 등 부가 정보

#### 4.6 ai_jobs 테이블

```sql
CREATE TABLE public.ai_jobs (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES public.users(id),
  map_id           UUID REFERENCES public.maps(id),
  job_type         VARCHAR(30) NOT NULL,   -- 'generate' | 'expand' | 'summarize'
  prompt           TEXT NOT NULL,
  result_markdown  TEXT,
  model            VARCHAR(100),
  tokens_used      INTEGER,
  status           VARCHAR(20) NOT NULL DEFAULT 'pending',
  created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 5. 동작 방식 (How)

#### 5.1 사용자 동작

* 툴바 `AI` 메뉴 → 기능 선택
* AI-01: 주제 입력 → 생성 → 맵에 삽입
* AI-02: 노드 선택 → `AI 확장` → 결과 프리뷰 → 수락/거부
* AI-03/04: Thread 패널 → 요약/추출 버튼 → 결과 검토

#### 5.2 시스템 처리 흐름

```
사용자 요청
    │
    ▼
POST /ai/generate | /ai/expand | /ai/summarize
    │
    ▼
ai_jobs INSERT (status: pending)
    │
    ▼
BullMQ Worker — LLM API 호출 (Claude / GPT)
    │
    ▼
result_markdown 저장 (ai_jobs.result_markdown)
    │
    ▼
WebSocket 또는 Polling → 클라이언트 결과 수신
    │
    ▼
Node Tree 파싱 → Document Store 반영
```

#### 5.3 Solo-only 정책

* 협업 중(접속자 2명 이상): AI 기능 비활성화 + 안내 배너 표시
* 단독 편집 모드(접속자 1명)에서만 AI 기능 허용

### 6. 규칙 (Rule)

* AI 기능은 단독 편집 모드에서만 활성화
* AI 생성 결과는 즉시 적용하지 않고 프리뷰 단계를 거쳐 수락/거부
* 프롬프트 최대 길이: 2000자
* 결과 노드 최대 생성 수: 50개 (1회 AI-01/02 요청 기준)
* API 오류 시 3회 재시도 후 오류 메시지 표시

---

### 7. 예외 / 경계 (Edge Case)

* **LLM API 타임아웃**: 30초 초과 시 오류 메시지 + 재시도 버튼
* **빈 프롬프트**: 입력 폼 유효성 검사로 차단
* **협업 중 AI 시도**: 안내 메시지 (`협업 중에는 AI 기능을 사용할 수 없습니다`)
* **결과 파싱 실패**: 원본 Markdown 텍스트를 단일 노드 note에 저장
