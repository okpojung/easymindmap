### 4. 기능 정의 (What)

#### 4.1 AI-01 Mindmap 자동 생성

```text
┌──────────────────────────────────────────────────────┐
│  AI Mindmap 생성                                      │
│  주제 또는 설명을 입력하세요:                              │
│  ┌────────────────────────────────────┐               │
│  │ Linux 서버 구축 절차                │               │
│  └────────────────────────────────────┘               │
│  [ 생성하기 ]                                          │
└──────────────────────────────────────────────────────┘
```

* 프롬프트 → `POST /ai/generate` → `ai_jobs` row INSERT (status: pending)
* BullMQ Worker가 LLM 호출 → Markdown outline 반환
* Markdown outline → Node Tree 파싱 → Document Store 반영

**레이아웃 및 연결선(Edge) 자동 결정 규칙**

AI가 맵을 생성할 때 레이아웃을 명시하지 않으면 기본값(`radial-bidirectional`)이 적용된다.  
레이아웃이 결정되면 연결선 타입이 **자동으로** 아래 규칙에 따라 결정된다.

| AI 생성 레이아웃 | Edge 타입 (자동 결정) | 연결선 형태 |
|---|---|---|
| `radial-bidirectional` (기본값) | `curve-line` | Cubic Bezier **곡선** |
| `radial-right` / `radial-left` | `curve-line` | Cubic Bezier **곡선** |
| `tree-*` / `hierarchy-*` / `process-tree-*` | `tree-line` | **직각선 (Orthogonal)** |
| `freeform` / `kanban` | `tree-line` | **직각선 (Orthogonal)** |

> **규칙 요약**: 방사형(Radial) → 곡선 / 나머지 모든 레이아웃 → 직각선(Orthogonal)  
> 연결선은 사용자가 직접 선택하지 않으며, 레이아웃 전환 시 자동으로 변경된다.  
> 참조: `docs/03-editor-core/edge-policy.md §3`

#### 4.2 AI-02 노드 확장

* 선택 노드 우클릭 > `AI 확장` 또는 툴바 AI 버튼
* 선택 노드 text + 상위 2레벨 컨텍스트 → LLM 전송
* 반환된 자식 아이디어 → 선택 노드 하위에 신규 자식 노드 생성

#### 4.3 AI-03 Thread 요약

* Node Thread 패널 상단 `[AI 요약]` 버튼
* Thread 메시지 전체 → LLM 요약 프롬프트
* 결과: 핵심 논점 / 결정 사항 / 미결 사항 3구분 출력
* 요약 결과는 노드 Note에 저장하거나 별도 팝업으로 표시

#### 4.4 AI-04 태스크 추출

* Thread 요약과 연계 또는 독립 실행
* 댓글 텍스트 분석 → Action Item · 담당자 후보 · 기한 후보 목록 반환
* 사용자 검토 후 개별 승인/거부 선택

#### 4.5 AI-05 태스크 노드 생성

* 승인된 태스크 후보 → 선택 노드의 자식 TODO 노드 일괄 생성
* 노드 text = 태스크명, note = 기한·담당자 등 부가 정보

#### 4.6 ai_jobs 테이블

```sql
CREATE TABLE public.ai_jobs (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES public.users(id),
  map_id           UUID REFERENCES public.maps(id),
  job_type         VARCHAR(30) NOT NULL,   -- 'generate' | 'expand' | 'summarize'
  prompt           TEXT NOT NULL,
  result_markdown  TEXT,
  model            VARCHAR(100),
  tokens_used      INTEGER,
  status           VARCHAR(20) NOT NULL DEFAULT 'pending',
  created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 13. AI 생성 입력 스펙 (AIGenerateRequest)

AI Mindmap 생성 요청 시 전달되는 파라미터 타입 정의.

```typescript
type AIGenerateRequest = {
  prompt: string;                        // 사용자 질문 (필수)
  maxDepth?: number;                     // 최대 depth — 기본값: 3
  maxChildrenPerNode?: number;           // 노드당 최대 자식 수 — 기본값: 5
  language?: 'ko' | 'en' | 'auto';      // 응답 언어 — 기본값: auto
  layoutType?: LayoutType;              // 생성 시 적용할 레이아웃 — 기본값: 'radial-bidirectional'
};
```

* `maxDepth`와 `maxChildrenPerNode`는 System Prompt 내 플레이스홀더로 LLM에 전달된다.
* `language`가 `'auto'`인 경우 LLM이 프롬프트 언어를 감지하여 동일 언어로 응답한다.
* `layoutType`이 지정되지 않으면 `'radial-bidirectional'`(곡선 edge)이 기본 적용된다.
* 서버는 `layoutType`으로부터 `edgeType`을 자동 결정한다 (`curve-line` 또는 `tree-line`).  
  이 값은 클라이언트에 반환되어 렌더러가 올바른 연결선을 그리도록 한다.

### 14. LLM System Prompt 명세

`POST /ai/generate` 호출 시 LLM에 전달되는 System Prompt 전문.

```
You are a mindmap generator.
Given a topic or question, generate a structured mindmap in Markdown format.

Rules:
- Use # for root topic
- Use ## for level 1 children
- Use ### for level 2 children
- Maximum depth: {maxDepth}
- Maximum children per node: {maxChildrenPerNode}
- Response ONLY in Markdown, no explanation text
- Keep node text concise (under 10 words per node)
- Language: {language}
```

* `{maxDepth}`, `{maxChildrenPerNode}`, `{language}` 는 요청 파라미터로 치환된다.
* LLM은 설명 텍스트 없이 Markdown 구조만 반환해야 한다.

### 15. Markdown → NodeTree 파서 알고리즘

LLM 응답 Markdown을 Node Tree로 변환하는 파서 명세.

```typescript
function parseMarkdownToNodes(markdown: string): NodeObject[] {
  // # → depth 0 (root 노드)
  // ## → depth 1 (1단계 자식)
  // ### → depth 2 (2단계 자식)
  // #### → depth 3 (3단계 자식)
  // - 또는 * 로 시작하는 항목 → 현재 depth의 자식으로 처리
}
```

#### 파싱 규칙 요약

| Markdown 패턴 | 매핑 depth | 비고 |
| ------------ | --------- | ---- |
| `#` (H1)     | 0         | 루트 노드 |
| `##` (H2)    | 1         | 1단계 자식 |
| `###` (H3)   | 2         | 2단계 자식 |
| `####` (H4)  | 3         | 3단계 자식 |
| `- ` / `* `  | 현재 depth의 자식 | 리스트 항목 |

#### 파싱 실패 처리

* 구조 파싱에 실패한 경우 원본 Markdown 텍스트를 단일 노드의 note에 저장한다.
* `ai_jobs.status` = `'parse_error'`로 기록한다.

## MVS 구현 — AI 마인드맵 생성 (2026-07)

서버 없이 브라우저에서 동작하는 1차 구현. 설계 원칙(사용자 결정):

> **웹 채팅에서 질문하고 답변을 받는 것과 똑같이** — 프롬프트를 입력하면
> AI의 답변(EMM Markdown)이 **그대로 맵으로 변환**된다. 구 설계의
> "최대 깊이/자식 수" 옵션은 폐기 — 구조는 EMM 프롬프트 템플릿 규칙과
> AI가 결정한다.

### 구성 (AITab — AI 생성 | AI 설정 2뷰)

- **AI 생성** (`AITab.tsx` GenerateView)
  - AI 선택: Anthropic(Claude) / OpenAI(ChatGPT) / Google(Gemini) —
    키 미등록 회사는 "키 미등록" 표시, 등록 전에는 생성 버튼 비활성
  - 생성 유형(선택): 기본 / 기술 절차형 / 회의록형 / WBS·계획형 /
    문서·기사 요약형 — EMM 템플릿에 용도별 추가 지시를 덧붙인다
    (emm-prompt-templates.md §3과 동일, `emmSystemPrompt.ts`)
  - 프롬프트 입력 → [AI로 맵 생성] → 확인(현재 맵 닫기 — Ctrl+Z로
    복귀 가능) → `parseEmm`으로 맵 변환 → 열기(방사형·오른쪽, 루트 선택)
  - 최근 생성 기록 10건 (클릭 = 프롬프트 재입력)
  - 오류는 사람이 읽을 수 있는 메시지로 표시 (상태코드 + API 메시지)
- **AI 설정** (SettingsView)
  - 회사별 API 키 + 모델 등록 (기본 모델: claude-sonnet-5 /
    gpt-4o / gemini-2.5-flash — 자유 수정. 2026-07: 소형 mini/2.0
    기본값이 웹 채팅 대비 답변이 짧던 원인이라 표준 모델로 상향 —
    예전 기본값을 그대로 쓰던 사용자는 자동 승격, 직접 고른 모델은
    보존)
  - **EMM 프롬프트 템플릿(시스템 프롬프트)** — 질문할 때 **항상
    기본으로** AI에게 함께 전달된다. 열람·수정 가능, "기본 템플릿
    복원" 버튼 (기본값 = emm-prompt-templates.md §2 공통 코어)
- 툴바의 [AI 생성] 버튼 = AI 탭 열기

### 개선 (2026-07 — 사용자 피드백 4건)

1. **사용 우선순위**: 키를 여러 개 등록했을 때 어떤 AI를 먼저 쓸지의
   순서(1·2·3, ▲▼로 조정). 생성 화면의 AI 선택 기본값은 **'자동 —
   우선순위 순'**으로, 순서에서 키가 등록된 첫 회사를 사용한다
   (수동 선택도 가능).
2. **모델 선택**: 회사별 알려진 모델 목록(select — 기본 표시:
   Anthropic claude-sonnet-5/opus-4-8/haiku-4-5 · OpenAI gpt-4o/
   4o-mini/4.1/4.1-mini · Gemini 2.5-flash/2.5-pro/2.0-flash/1.5-pro)
   + '직접 입력…'으로 목록에 없는 새 모델 사용.
3. **상세함 규칙(템플릿 11번)**: "답변의 내용과 상세함은 웹 채팅과
   똑같아야 한다 — 구조화를 이유로 요약·단축 금지, 정보량 유지".
   emm-prompt-templates.md와 내장본 동기. 기존 사용자의 저장된 기본
   템플릿은 자동 마이그레이션(직접 수정본은 보존).
4. **API 키 발급 도움말**: 회사별 "키 발급 방법 (처음이신가요?)" 접이식
   안내 — 발급 페이지 URL(콘솔 바로가기) + IT 초보자 단계별 절차
   (가입→메뉴→키 생성→복사→결제 안내).
