### 1. 기능 목적

* 맵 편집 중 **AI를 활용하여 노드 생성·확장·요약·태스크 추출**을 자동화하는 기능
* Claude / GPT 등 LLM 기반으로 마인드맵 구조 자동 생성 및 확장
* 협업 스레드 요약과 할일 노드 생성으로 회의·논의 결과를 즉시 구조화

### 4. 기능 정의 (What)

#### 4.1 AI-01 Mindmap 자동 생성

```text
┌──────────────────────────────────────────────────────┐
│  AI Mindmap 생성                                      │
│  주제 또는 설명을 입력하세요:                              │
│  ┌────────────────────────────────────┐               │
│  │ Linux 서버 구축 절차                │               │
│  └────────────────────────────────────┘               │
│  [ 생성하기 ]                                          │
└──────────────────────────────────────────────────────┘
```

* 프롬프트 → `POST /ai/generate` → `ai_jobs` row INSERT (status: pending)
* BullMQ Worker가 LLM 호출 → Markdown outline 반환
* Markdown outline → Node Tree 파싱 → Document Store 반영

#### 4.2 AI-02 노드 확장

* 선택 노드 우클릭 > `AI 확장` 또는 툴바 AI 버튼
* 선택 노드 text + 상위 2레벨 컨텍스트 → LLM 전송
* 반환된 자식 아이디어 → 선택 노드 하위에 신규 자식 노드 생성

#### 4.3 AI-03 Thread 요약

* Node Thread 패널 상단 `[AI 요약]` 버튼
* Thread 메시지 전체 → LLM 요약 프롬프트
* 결과: 핵심 논점 / 결정 사항 / 미결 사항 3구분 출력
* 요약 결과는 노드 Note에 저장하거나 별도 팝업으로 표시

#### 4.4 AI-04 태스크 추출

* Thread 요약과 연계 또는 독립 실행
* 댓글 텍스트 분석 → Action Item · 담당자 후보 · 기한 후보 목록 반환
* 사용자 검토 후 개별 승인/거부 선택

#### 4.5 AI-05 태스크 노드 생성

* 승인된 태스크 후보 → 선택 노드의 자식 TODO 노드 일괄 생성
* 노드 text = 태스크명, note = 기한·담당자 등 부가 정보

#### 4.6 ai_jobs 테이블

```sql
CREATE TABLE public.ai_jobs (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES public.users(id),
  map_id           UUID REFERENCES public.maps(id),
  job_type         VARCHAR(30) NOT NULL,   -- 'generate' | 'expand' | 'summarize'
  prompt           TEXT NOT NULL,
  result_markdown  TEXT,
  model            VARCHAR(100),
  tokens_used      INTEGER,
  status           VARCHAR(20) NOT NULL DEFAULT 'pending',
  created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 5. 동작 방식 (How)

#### 5.1 사용자 동작

* 툴바 `AI` 메뉴 → 기능 선택
* AI-01: 주제 입력 → 생성 → 맵에 삽입
* AI-02: 노드 선택 → `AI 확장` → 결과 프리뷰 → 수락/거부
* AI-03/04: Thread 패널 → 요약/추출 버튼 → 결과 검토

#### 5.2 시스템 처리 흐름

```
사용자 요청
    │
    ▼
POST /ai/generate | /ai/expand | /ai/summarize
    │
    ▼
ai_jobs INSERT (status: pending)
    │
    ▼
BullMQ Worker — LLM API 호출 (Claude / GPT)
    │
    ▼
result_markdown 저장 (ai_jobs.result_markdown)
    │
    ▼
WebSocket 또는 Polling → 클라이언트 결과 수신
    │
    ▼
Node Tree 파싱 → Document Store 반영
```

#### 5.3 Solo-only 정책

* 협업 중(접속자 2명 이상): AI 기능 비활성화 + 안내 배너 표시
* 단독 편집 모드(접속자 1명)에서만 AI 기능 허용

### 6. 규칙 (Rule)

* AI 기능은 단독 편집 모드에서만 활성화
* AI 생성 결과는 즉시 적용하지 않고 프리뷰 단계를 거쳐 수락/거부
* 프롬프트 최대 길이: 2000자
* 결과 노드 최대 생성 수: 50개 (1회 AI-01/02 요청 기준)
* API 오류 시 3회 재시도 후 오류 메시지 표시

---

### 7. 예외 / 경계 (Edge Case)

* **LLM API 타임아웃**: 30초 초과 시 오류 메시지 + 재시도 버튼
* **빈 프롬프트**: 입력 폼 유효성 검사로 차단
* **협업 중 AI 시도**: 안내 메시지 (`협업 중에는 AI 기능을 사용할 수 없습니다`)
* **결과 파싱 실패**: 원본 Markdown 텍스트를 단일 노드 note에 저장

### 13. AI 생성 입력 스펙 (AIGenerateRequest)

AI Mindmap 생성 요청 시 전달되는 파라미터 타입 정의.

```typescript
type AIGenerateRequest = {
  prompt: string;                        // 사용자 질문 (필수)
  maxDepth?: number;                     // 최대 depth — 기본값: 3
  maxChildrenPerNode?: number;           // 노드당 최대 자식 수 — 기본값: 5
  language?: 'ko' | 'en' | 'auto';      // 응답 언어 — 기본값: auto
};
```

* `maxDepth`와 `maxChildrenPerNode`는 System Prompt 내 플레이스홀더로 LLM에 전달된다.
* `language`가 `'auto'`인 경우 LLM이 프롬프트 언어를 감지하여 동일 언어로 응답한다.

---

### 14. LLM System Prompt 명세

`POST /ai/generate` 호출 시 LLM에 전달되는 System Prompt 전문.

```
You are a mindmap generator.
Given a topic or question, generate a structured mindmap in Markdown format.

Rules:
- Use # for root topic
- Use ## for level 1 children
- Use ### for level 2 children
- Maximum depth: {maxDepth}
- Maximum children per node: {maxChildrenPerNode}
- Response ONLY in Markdown, no explanation text
- Keep node text concise (under 10 words per node)
- Language: {language}
```

* `{maxDepth}`, `{maxChildrenPerNode}`, `{language}` 는 요청 파라미터로 치환된다.
* LLM은 설명 텍스트 없이 Markdown 구조만 반환해야 한다.

### 15. Markdown → NodeTree 파서 알고리즘

LLM 응답 Markdown을 Node Tree로 변환하는 파서 명세.

```typescript
function parseMarkdownToNodes(markdown: string): NodeObject[] {
  // # → depth 0 (root 노드)
  // ## → depth 1 (1단계 자식)
  // ### → depth 2 (2단계 자식)
  // #### → depth 3 (3단계 자식)
  // - 또는 * 로 시작하는 항목 → 현재 depth의 자식으로 처리
}
```

#### 파싱 규칙 요약

| Markdown 패턴 | 매핑 depth | 비고 |
| ------------ | --------- | ---- |
| `#` (H1)     | 0         | 루트 노드 |
| `##` (H2)    | 1         | 1단계 자식 |
| `###` (H3)   | 2         | 2단계 자식 |
| `####` (H4)  | 3         | 3단계 자식 |
| `- ` / `* `  | 현재 depth의 자식 | 리스트 항목 |

#### 파싱 실패 처리

* 구조 파싱에 실패한 경우 원본 Markdown 텍스트를 단일 노드의 note에 저장한다.
* `ai_jobs.status` = `'parse_error'`로 기록한다.

### 16. AI 생성 제한 정책

| 항목 | 제한값 |
| ---- | ------ |
| 최대 depth | 4 |
| 노드당 최대 자식 수 | 7 |
| 노드 텍스트 최대 길이 | 50자 |
| 프롬프트 최대 길이 | 500자 |
| 분당 요청 제한 | 10회 (사용자당) |
| 1회 최대 생성 노드 수 | 50개 |

* 위 제한을 초과하는 요청은 API에서 422 응답으로 차단한다.
* 분당 요청 제한 초과 시 429 응답 반환 및 재시도 대기 시간 안내.

### 17. 협업 임계값 및 AI 제한 정책

동시 접속자 수 기준으로 AI 기능 활성화 여부를 결정한다.

#### 임계값 정의

| 접속자 수 | AI 기능 상태 |
| -------- | ----------- |
| 1명 (단독 편집) | AI 전 기능 활성화 |
| 2명 이상 (협업) | AI 전 기능 비활성화 |

#### 비활성화 대상

협업 중(접속자 수 ≥ 2)일 때 아래 기능이 모두 제한된다.

* AI Generate (AI-01)
* AI Expand (AI-02)
* AI Workflow Generate (→ `19-ai-workflow.md`)
* map 구조를 변경하는 모든 AI 반영 기능

#### UI 처리 원칙

* AI 버튼을 `disabled` 상태로 표시한다.
* 안내 메시지를 배너 또는 툴팁으로 표시한다.
  * 예: `"협업 중에는 AI 자동 생성/확장 기능을 사용할 수 없습니다."`
* 접속자가 1명으로 복귀하면 AI 버튼을 즉시 활성화한다.
