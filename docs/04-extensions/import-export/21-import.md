### 1. 기능 목적

* 외부 Markdown 파일을 **마인드맵 노드 트리로 변환하여 가져오는** 기능
* Obsidian, Notion, VS Code 등에서 작성한 아웃라인 문서를 즉시 마인드맵화
* 기존 텍스트 기반 계획서·절차서를 마인드맵으로 빠르게 전환

### 4. 기능 정의 (What)

#### 4.1 파싱 모드 선택

| 모드          | 적용 조건                     | 설명                   |
| ----------- | ------------------------- | -------------------- |
| Hybrid Mode  | 기본값 (권장)                 | 헤딩 구조화 + 리스트 타입별 분리 적용 |
| Document Mode | 명시적 선택 시                 | 모든 리스트를 내부 콘텐츠로 강제 유지 |
| Outline Mode | 명시적 선택 시                 | 모든 리스트를 자식 노드로 강제 구조화 |

#### 4.2 Hybrid Mode 변환 규칙 (Smart Hybrid Mode)

```text
# Root 제목         → Root 노드 (depth 0)
## 섹션 A           → 자식 노드 (depth 1)
### 항목 A-1        → 자식 노드 (depth 2)

- 순서 없는 목록      → 상위 노드의 note/text로 병합 (구조화 안함)
1. 순서 있는 목록     → 상위 노드의 자식 노드로 분리
- [ ] 체크박스 목록   → 상위 노드의 자식 TODO 노드로 분리

헤딩 사이 paragraph → 상위 노드의 note로 저장
코드 블록           → 상위 노드의 note 코드 블록으로 저장
인라인 태그 [tag]    → 해당 노드에 태그 자동 할당 (태그 존재 시)
```

#### 4.3 Document Mode 변환 규칙

```text
빈 줄 구분 단락 → 각각 독립 자식 노드
모든 리스트     → 노드 내부 콘텐츠로 강제 유지 (자식 노드 생성 안함)
파일명          → Root 노드 text
```

#### 4.4 가져오기 미리보기 UI

```text
┌─────────────────────────────────────────────────────┐
│  가져오기 미리보기                          [✕]       │
├─────────────────────────────────────────────────────┤
│  파일: linux-setup.md                                │
│  모드: Hybrid Mode     [변경]                         │
│  노드 수: 24개                                        │
│                                                     │
│  📁 Linux 서버 구축                                   │
│    ├── 📄 패키지 관리                                 │
│    │     ├── 📄 APT 업데이트                           │
│    │     └── 📄 Nginx 설치                            │
│    └── 📄 보안 설정                                   │
│          └── 📄 방화벽 설정                            │
│                                                     │
│  [ 취소 ]          [ 새 맵으로 가져오기 ] [ 현재 맵에 추가 ] │
└─────────────────────────────────────────────────────┘
```

### 5. 동작 방식 (How)

#### 5.1 사용자 동작

* 상단 메뉴 > `가져오기` > `Markdown 파일`
* 파일 선택 다이얼로그 → `.md` 파일 선택 (또는 드래그 앤 드롭)
* 파싱 모드 자동 선택 (헤딩 구조 감지)
* 미리보기 화면에서 노드 트리 구조 확인
* `새 맵으로 가져오기` 또는 `현재 맵에 추가` 선택 후 확정

#### 5.2 시스템 처리 흐름

```
파일 선택 (File input / Drag & Drop)
    │
    ▼
클라이언트 파싱 (Markdown → AST)
    │
    ▼
파싱 모드 자동 감지 (헤딩 수 기준)
    │
    ▼
AST → NodeTree 변환
    │
    ▼
미리보기 렌더링
    │
    ▼ (사용자 확정)
POST /maps/{mapId}/import { nodes: NodeTree }
  또는
POST /maps { title, nodes: NodeTree }  ← 새 맵 생성
    │
    ▼
Document Store 반영 + Auto Save
```

### 6. 규칙 (Rule)

* 지원 파일 형식: `.md`, `.markdown`
* 파일 최대 크기: 5MB
* 노드 최대 생성 수: 500개 (1회 가져오기 기준)
* 헤딩 최대 깊이: H6 (`######`) → depth 5 (이하 flat 처리)
* 중복 태그 [tag]는 자동 매칭 (기존 태그가 없으면 신규 생성)

---

### 7. 예외 / 경계 (Edge Case)

* **지원하지 않는 파일 형식**: `.md`, `.markdown` 외 파일 선택 시 오류 안내
* **파일 크기 초과**: 5MB 초과 시 오류 메시지
* **노드 수 초과**: 500개 초과 경고 → 분할 가져오기 안내
* **빈 파일**: "가져올 내용이 없습니다" 안내 메시지
* **파싱 실패**: 원본 텍스트 전체를 단일 노드 note에 저장

## MVS 구현 — MD 완전 변환 (2026-07)

일반 MD 불러오기(importMarkdown.ts)는 견출·리스트만이 아니라 **문서의
모든 내용**을 맵으로 변환한다:

| MD 요소 | 변환 결과 |
|---|---|
| `# 제목` | 루트(중심 주제) · 맵 제목 — **파일의 첫 H1만** |
| **제목 이후의 `#` 견출(본문 H1)** | **2레벨 견출 노드** — ChatGPT 대화 내보내기처럼 본문 견출에 #을 쓰는 파일에서 견출이 사라지지 않는다. 본문 H1이 있으면(h1Mode) `##`은 3레벨, `###`은 4레벨…로 한 단계씩 내려 계층 보존 |
| **첫 견출 전의 인용문(>)·문단·표** | **루트의 노트** (머리말 처리 — 문단/표 노트) |
| `##`~`######` 견출 | 2~6레벨 노드 (파일 순서 그대로 · h1Mode면 한 단계 아래) |
| `-`/`*`/`+` 리스트 | 마지막 견출의 하위 노드 (들여쓰기 2칸 = 한 단계) |
| `1.` / `1)` 순번 리스트 | 하위 노드 — **번호("1. ")를 텍스트에 그대로 유지** |
| **들여쓰기 없는 순번 항목 = "절 머리"** | 다음 순번/견출 전까지의 **문단이 그 하위로** 붙는다 (ChatGPT 답변처럼 견출 없이 `1. 제목` + 본문 문단으로 쓰는 문서 — 코드 펜스는 절의 코드 노트가 된다) |
| **문단 뒤의 불릿(`-`,`*`)** | 그 **문단의 하위**로 붙는다 (예: "Apache 설정 파일 수정" 문단 아래 `* DocumentRoot: …`) |
| 백슬래시 이스케이프(`1\.` `\-` 등) | 원래 문자로 해제해 노드 텍스트에 저장 |
| **리스트 항목의 들여쓴 연속 줄** | 그 항목 노드의 **추가 줄(\n)로 병합** (예: `1. 원본 파일 복사` 아래의 "예: 견적서.xlsx → …") |
| 일반 문단 | 마지막 견출의 하위 노드 (연속 줄 = 한 노드, 줄바꿈 유지) |
| `\| 표 \|` (연속 파이프 행) | **직전 노드의 표 노트** — 구분선(`\|---\|`)·앞뒤 파이프 제거 후 노트 뷰어가 격자 표로 렌더링 (노드 내용 아님) |
| `[라벨](url)` 하이퍼링크 | 노드 텍스트에는 **라벨만** 남고 URL은 **노드의 링크(🔗)로 첨부** ([![배지](img)](url) 중첩도 해체). 노트(인용문) 안의 링크는 원문 유지 — **노트 뷰어가 클릭 가능한 `<a>`로 렌더링** |
| `> 인용문` | 현재 노드의 **문단 노트** |
| ``` 코드 펜스 ``` | 현재 노드의 **코드 노트** (언어 표기 유지) |
| 인라인 강조(`**` `*` `~~` `__` `==`) | 노드 텍스트에 그대로 → 인라인 마커 렌더링 |
| `---` 수평선 | 무시 |

- **순서 보장**: 파서는 순차 처리 — 2레벨 이하 모든 노드가 파일에 적힌
  순서 그대로 배치된다.
- MD 내보내기(exportMarkdown.ts)도 대칭: 문단 노트 → `>` 인용문,
  코드 노트 → 펜스, **표 노트 → 파이프 표(헤더 구분선 포함)**,
  노드 링크 → `🔗 [라벨](url)`, 루트 노트 → 제목 아래 인용문·표·펜스로
  내보내 왕복된다.
- **검증 코퍼스**: 전자영수증 보고서(194노드)·ChatGPT 대화
  내보내기(299노드)·README형·회의록(본문 H1 다수)·깊은 견출·머리말만
  있는 문서 — 6종 파일로 변환·왕복 E2E 상시 검증 (e2e31). WEB/WAS
  설치 대화(견출 없는 순번 절 + 문단 + 불릿 + 코드 펜스)는 e2e32.
- **알려진 한계(규칙 기반)**: 절의 마지막 마무리 문단("이제 … 할 수
  있습니다")도 절의 자식이 된다(절 종료 신호가 없음). `**굵은 소제목**`
  문단은 아직 절 머리로 취급하지 않는다 — 필요해지면 "짧은 전부-굵게
  문단 = 소제목" 규칙 추가 검토.

## MVS 구현 — 새 맵 메뉴 재편 + 템플릿 선택 단계 (2026-07)

'새 맵' 메뉴(NewMapPanel) 구성:

- **새 맵 만들기** — 기본 템플릿 '트리-진행트리맵' 골격으로 시작
  (2026-07 지정): 중심 주제 + 주제 1~3 + 각 하위 주제 2 + 내용 (4레벨).
  레벨별 레이아웃 = 1레벨 트리·오른쪽 → 2레벨 진행트리·오른쪽 →
  3레벨 트리·오른쪽 → 4레벨 진행트리·오른쪽
  (documentStore.newMap — 노드의 layoutType은 그 노드의 "자식" 배치)
- **서버 맵 불러오기** — [서버 연결 예정]
- **Local 파일 불러오기**
  - MD 파일 불러오기 (일반 MD + EasyMindMap 생성 MD)
  - HTML 파일 불러오기 (EasyMindMap 생성 HTML만)
  - ZIP 파일 불러오기 (EasyMindMap HTML/MD + files/ 첨부)
- **등록된 템플릿에서 시작** — 내 템플릿 + 라이브러리 템플릿 골격

새 맵을 만들거나 파일을 불러온 **직후 "적용할 템플릿 선택" 단계**가
나타난다 — 내 템플릿·라이브러리 템플릿 중 골라 속성(레이아웃·스타일·
맵 설정)만 입히거나(applyTemplateStyles) '건너뛰기(기본 그대로)'.

### 템플릿 라이브러리 (libraryTemplates.ts)

기본 제공 7종 — 실제 적용/시작 가능한 맵을 가진다:
**트리-진행트리맵(기본 템플릿 — 1레벨 트리 → 2레벨 진행트리 → 3레벨
트리 → 4레벨 진행트리, 목록 첫 번째·'새 맵 만들기' 골격과 동일)** ·
진행트리-트리맵(1레벨 진행트리 + 2레벨 트리) · 브레인스토밍(방사형·양쪽)
· 제품 로드맵 · WBS 프로젝트(계층형) · Kanban 보드 · 회의록.
[서버 연결 예정] 시스템 관리자 템플릿 카탈로그로 이관.

### 템플릿 '적용'의 레벨 레이아웃 복원 (userTemplates.applyTemplateStyles)

템플릿의 depth별 **대표 서브트리 레이아웃**(그 depth 첫 노드의
layoutType)을 현재 맵의 같은 depth 모든 노드에 적용한다 (없으면 해제 —
스타일과 동일한 "템플릿이 곧 정답" 규칙). 루트 layoutType도 복원.
2026-07: MD로 불러온 맵에 기본트리맵을 적용해도 전체가 진행트리로만
바뀌고 2레벨 트리 레이아웃이 재현되지 않던 문제의 수정.

### 관련 수정 (같은 커밋)

- **긴 무공백 토큰 줄바꿈**: 줄 중간에 등장한 폭 초과 단어도 문자
  단위로 분해 (sizeNodeForText — '과기정통부·…·나이스정보통신이' 같은
  토큰이 테두리를 넘던 문제).
- **대문자 라틴 폭 보정**: KISTI·NHN·POS 등 약어가 0.55배율로
  과소측정되던 것을 0.72로 (sizeNodeForText·mdTable 공통).
- **템플릿 '적용' = 속성만**: 현재 맵의 내용은 유지하고 템플릿의
  레벨별 대표 스타일 + 맵 설정(레벨 폰트·레이아웃) + 전체 레이아웃·
  간격만 입힌다 (userTemplates.applyTemplateStyles). 맵 전체 교체는
  '새 맵 > 이 템플릿으로 시작'으로만.
