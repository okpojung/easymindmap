# Markdown ↔ 맵 변환 규칙 명세 (markdown-export.md)

* 문서 버전: v1.2
* 최초 작성: 2026-07 (MVS)
* **위상**: 이 문서는 **EMM(EasyMindMap Markdown) 스펙의 구현 규칙
  상세서**다 — 포맷의 설계 원칙·적합성·버전 정책은 `emm-spec.md`가
  규범이고, 변환 표 단위의 현행 규칙은 이 문서가 규범이다.
* **이 문서는 EasyMindMap의 핵심 기능인 "MD 파일 ↔ 맵" 변환의 단일
  기준 명세다. `importMarkdown.ts` / `exportMarkdown.ts`의 변환 규칙이
  바뀔 때마다 반드시 이 문서에 반영한다.**
* 구현: `apps/frontend/src/utils/importMarkdown.ts` (MD → 맵),
  `apps/frontend/src/export/exportMarkdown.ts` (맵 → MD)
* 관련: `import-export/20-export.md`(내보내기 UI·패키징),
  `import-export/21-import.md`(불러오기 UI·모드),
  `import-export/22-map-file-meta.md`(메타데이터 왕복)

## 1. MD → 맵 변환 규칙 (importMarkdown.parseMarkdownToMap)

파일을 위에서 아래로 순차 처리한다 — **2레벨 이하 모든 노드는 파일에
적힌 순서 그대로** 배치된다.

### 1.1 견출(heading)

| 입력 | 결과 |
|---|---|
| 파일의 **첫 H1**(`# 제목`) | 루트(중심 주제) 텍스트 + 맵 제목 |
| **제목 이후의 H1** (본문 `#`) | 2레벨 견출 노드 — ChatGPT 내보내기처럼 본문 견출에 `#`을 쓰는 파일에서 견출이 사라지지 않는다 |
| `##` ~ `######` | 2~6레벨 노드 |
| **h1Mode** (사전 스캔에서 제목 외 본문 H1 발견 시) | `#`=2레벨, `##`=3레벨 … 한 단계씩 내려 `#`/`##` 계층을 보존 |

### 1.2 리스트

| 입력 | 결과 |
|---|---|
| `-` `*` `+` 불릿 | 마지막 견출(또는 아래 1.4의 기준)의 하위 노드. 들여쓰기 2칸/탭 = 한 단계 |
| `1.` `1)` 순번 | 하위 노드 — **번호("1. ")를 노드 텍스트에 그대로 유지** |
| **들여쓰기 없는 순번 항목 = "절 머리"** | 다음 순번/견출 전까지의 문단이 그 하위로 붙는다 (견출 없이 `1. 제목` + 본문 문단으로 쓰는 문서 — ChatGPT 답변 형식) |
| 리스트 항목의 **들여쓴 연속 줄** | 그 항목 노드의 추가 줄(`\n`)로 병합 (예: `1. 원본 파일 복사` 아래 `   예: 견적서.xlsx → …`) |

### 1.3 문단

| 입력 | 결과 |
|---|---|
| 첫 견출 전의 문단·인용문·표 | **루트의 노트** (머리말 처리 — 문단/표 노트) |
| 일반 문단 (연속 줄) | 한 노드로 합쳐 마지막 견출의 하위 (줄바꿈 유지) |
| **절 머리(1.2) 뒤의 문단** | 그 절 노드의 **자식** |

### 1.4 문단 뒤의 불릿

문단 바로 뒤의 불릿(`-`,`*`)은 **그 문단 노드의 하위**로 붙는다.

```md
Apache 설정 파일 수정 (`/etc/httpd/conf/httpd.conf`)

* DocumentRoot: 웹 문서 루트 디렉터리 설정
* DirectoryIndex: 기본 웹 페이지 파일명 설정
```
→ "Apache 설정 파일 수정 …" 노드의 자식으로 DocumentRoot / DirectoryIndex.

### 1.5 표

| 입력 | 결과 |
|---|---|
| `\| 셀 \| 셀 \|` 연속 행 | **직전 노드의 "표 노트"** (노드 내용이 아님) — 구분선(`\|---\|`)과 앞뒤 파이프를 제거해 "셀 \| 셀" 줄 형식으로 저장. 노트 뷰어(⊞)가 격자 표로 렌더링하며 셀 안 인라인 마커·링크도 서식으로 표시 |
| 첫 견출 전의 표 | 루트의 표 노트 |

### 1.6 인용문·코드

| 입력 | 결과 |
|---|---|
| `> 인용문` | 현재 노드의 **문단 노트** (원문 유지 — 노트 뷰어가 인라인 마커·`[라벨](url)` 링크를 서식/앵커로 렌더링) |
| ``` 코드 펜스 ``` | 현재 노드의 **코드 노트** (언어 표기 유지). 절 머리 뒤의 펜스는 그 절 노드의 코드 노트 |

### 1.7 하이퍼링크·이미지

| 입력 | 결과 |
|---|---|
| `[라벨](url)` (노드가 되는 텍스트 안) | 텍스트에는 **라벨만** 남기고 URL은 **노드의 링크(🔗)** 로 첨부 (URL 중복 제거) |
| `[![배지](img)](url)` 중첩 | 대체 텍스트만 남기고 바깥 URL을 링크로 추출 |
| `![대체](경로)` 이미지 문법 | 대체 텍스트만 남김 |
| 노트(인용문) 안의 링크 | 원문 유지 — 노트 뷰어가 클릭 가능한 `<a>`로 렌더링 |

### 1.8 기타

| 입력 | 결과 |
|---|---|
| 인라인 강조 `**` `*` `~~` `__` `==` | 노드 텍스트에 그대로 저장 → 에디터가 인라인 마커로 렌더링 (마커 문자 숨김) |
| 백슬래시 이스케이프 (`1\.` `\-` 등) | 원래 문자로 해제 |
| `---` 수평선 | 무시 (절 머리·문단 앵커는 초기화) |

### 1.9 알려진 한계 (규칙 기반)

* 절의 마지막 마무리 문단("이제 … 할 수 있습니다")도 절의 자식이 된다
  — 절 종료 신호가 없음.
* `**굵은 소제목**` 문단은 아직 절 머리로 취급하지 않는다 — 필요 시
  "짧은 전부-굵게 문단 = 소제목" 규칙 추가 검토.

## 2. 맵 → MD 변환 규칙 (exportMarkdown) — 1과 왕복 대칭

| 맵 요소 | 출력 |
|---|---|
| 루트 | `# 제목` |
| 2~6레벨 노드 | `##` ~ `######` (7레벨+는 리스트 들여쓰기) |
| 노드 텍스트 줄바꿈 | 본문에서는 공백으로 합침 (원본은 메타데이터가 보존) |
| 노드 링크 | `🔗 [라벨](url)` — 다시 불러오면 노드 링크로 재추출 |
| 문단 노트 | `>` 인용문 |
| 코드 노트 | ``` 펜스 (언어 표기) |
| 표 노트 | 파이프 표 (헤더 다음 `\|---\|` 구분선 포함) |
| 루트 노트 | 제목 바로 아래 인용문·표·펜스 |
| 사진 | `![…](files/…)` + ZIP 패키징 (첨부 있으면 `제목.md + files/` ZIP) |
| 노드 인라인 사진 (`images[]` — 텍스트 중간, 기사 붙여넣기) | 노드 제목 아래 원문 **순서대로** `![…](…)` 나열 — MD에서 노드는 한 줄이라 "텍스트 중간" 위치(afterLine)는 표현하지 못하며, 위치 정보는 맵 메타데이터(easymindmap 주석)로만 복원된다. 사진은 붙여넣기 시점에 다운로드 내장(data URL)되므로 대부분 `files/img-N.*`로 ZIP 패키징된다 (CORS 차단 폴백 URL만 원본 주소 그대로) |
| 맵 메타데이터 | 파일 끝 주석 `easymindmap:v1:BASE64` — 다시 불러오면 스타일·노트·설정까지 복원 (22-map-file-meta.md) |
